## 5. 面向对象与 STL 评分证据

- [x] 封装：`GameMap`、`Player`、`WaterBubble` 等类的状态私有化，通过公开行为维护不变量。
- [x] 继承：`GameObject → Character → Player/Enemy → PatrolEnemy/HunterEnemy/BossEnemy`。
- [x] 多态：异构敌人存入 `std::vector<std::unique_ptr<Enemy>>`，统一调用虚函数更新、绘制、决策和受击。
- [x] STL 固定数据：`std::array` 保存地图格和 256 个键盘状态。
- [x] STL 动态数据：`std::vector` 保存水泡、水浪、道具、粒子、敌人和危险格。
- [x] STL 生命周期：`std::unique_ptr` 自动释放派生敌人，避免裸指针所有权。
- [x] STL 算法：`std::clamp`、`std::find`、`std::count`、`std::remove_if`、`std::min_element`。
- [x] 随机数据：`std::mt19937` 配合标准分布生成可复现地图、掉落和 AI 决策。
