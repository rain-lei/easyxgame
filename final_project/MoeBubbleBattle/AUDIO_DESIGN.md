# 《萌泡大作战》音频设计

## 1. 原创性

项目中的音乐和音效均由 `tools/generate_audio.py` 使用基础波形与原创旋律生成，不截取、不改编其他游戏或商业音乐。提交时保留生成脚本，便于说明素材来源和复现过程。

## 2. 音频清单

| 文件 | 场景 | 播放方式 |
| --- | --- | --- |
| `bgm_menu.wav` | 主菜单、角色选择、说明 | 循环播放，音量较低 |
| `bgm_game.wav` | 三个正式关卡 | 循环播放，暂停时同步暂停 |
| `sfx_menu_move.wav` | 菜单切换选项 | 单次播放 |
| `sfx_menu_confirm.wav` | 确认按钮 | 单次播放 |
| `sfx_bubble_place.wav` | 玩家成功放置水泡 | 单次播放 |
| `sfx_bubble_explode.wav` | 水泡释放水浪 | 单次播放；连锁时限制并发频率 |
| `sfx_power_up.wav` | 拾取强化道具 | 单次播放 |
| `sfx_hit.wav` | 玩家受伤 | 单次播放 |
| `sfx_level_clear.wav` | 关卡完成或最终通关 | 单次播放，暂时压低背景音乐 |

## 3. 技术参数

- 格式：PCM WAV。
- 采样率：44.1 kHz。
- 位深：16 bit。
- 声道：单声道。
- 背景音乐采用可循环的短曲，减少提交体积。
- C++ 中使用 Windows MCI 播放背景音乐，短音效使用 `PlaySound` 异步播放。

## 4. 音量原则

- 背景音乐作为陪衬，不遮盖讲解视频中的人声。
- 菜单确认、拾取和受伤音效清楚但不刺耳。
- 连锁爆炸限制相同音效触发频率，避免声音叠加失真。
- 暂停界面降低或暂停背景音乐；返回游戏后继续。

## 5. 重新生成

在项目根目录执行：

```powershell
python .\tools\generate_audio.py --output .\assets\audio
```

脚本结束时会输出每个 WAV 文件的时长、采样率和声道数，用于基本格式验收。
