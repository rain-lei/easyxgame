# 《萌泡大作战》5 分钟项目汇报录制稿

## 录制参数

- 画面：1280×720、30 FPS、横屏；录制鼠标指针。
- 声音：开启麦克风和系统声音，游戏音乐保持在不盖住人声的音量。
- 程序：Visual Studio 选择 `Release | x64`，按 `Ctrl+F5` 启动。
- 时长：目标 4 分 30 秒至 4 分 50 秒，留出片头、切换和失误余量。
- 隐私：关闭 QQ/微信通知，不展示 GitHub 密钥、邮箱密码和私人文件。
- 提交：剪辑完成后导出 720P，再按老师要求转换为 FLV。

## 分镜与讲解词

### 00:00—00:20 片头与身份

画面：报告封面或 Visual Studio 解决方案资源管理器，展示项目名和源代码目录。

讲解：

> 大家好，我是［姓名］，学号［学号］。我的面向对象课程实践项目是《萌泡大作战》，它是一款使用 C++、Visual Studio 和 EasyX 开发的 Q 版单人网格闯关游戏。下面我从玩法、界面、对象设计和测试四个方面进行演示。

### 00:20—00:55 主菜单与鼠标交互

画面：启动游戏；移动鼠标让按钮出现悬停状态；点击“操作说明”，再用鼠标返回；点击“开始游戏”。

讲解：

> 主菜单、操作说明、角色选择、暂停和结算界面都同时支持键盘和鼠标。人物原画不是单独放在素材目录里，而是实际用于主菜单、角色卡片和 HUD。背景音乐、按钮音效和战斗音效也由程序按场景切换。

验收点：鼠标悬停、点击、返回均正常；中文和人物图片无乱码、无黑底。

### 00:55—01:30 四名角色差异

画面：在角色选择页依次悬停或选择四张角色卡，最后选择一名角色进入游戏。

讲解：

> 四名角色不是只换颜色。蓝熊是全能型，初始两枚水泡；珊瑚兔移动最快；薄荷猫生命较少，但水泡范围更大；金毛犬有四点生命并自带一次护盾。角色生命、速度、水泡容量、威力和护盾集中在 `CharacterProfile` 中配置，因此调整平衡时不需要修改玩家类内部逻辑。

验收点：四张卡片的生命、移速、容量、范围或护盾数值确实不同。

### 01:30—02:25 移动、水泡和道具

画面：使用 WASD 或方向键连续移动、转过窄道；放置水泡并避开水浪；破坏木箱后拾取道具；展示 HUD 数值变化。

讲解：

> 地图采用十五乘十三的网格。可见人物可以覆盖通道，但碰撞只使用约二十乘二十像素的脚底区域，并在转角处进行有限的格心纠偏，所以移动不会因为头发或背包碰墙而卡住。行走动画采用回中、跨步、回中的六阶段播放，减少左右摆动。水泡在两点五秒后产生十字水浪，固定墙会阻挡水浪，木箱被破坏后终止传播，其他水泡会被连锁触发。四种原创图标分别表示范围、容量、速度和护盾，拾取后 HUD 会立即变化。

验收点：连续移动顺滑；至少成功放置一枚水泡并拾取一种道具。

### 02:25—03:05 敌人、暂停与 Boss

画面：展示巡逻敌人和追踪敌人；按 P 或 Esc 暂停，再用鼠标点击继续。若录制时间内无法正常推进到第三关，在剪辑中插入 `report/assets/game_level3_boss.png`，随后展示 `GameTypes.h` 中的 `BossMaxHealth`。

讲解：

> 普通敌人分为随机巡逻和追踪玩家两种。第三关首领“墨潮王”默认有十点生命，头顶和 HUD 都显示血条；血量可以在 `GameConfig::BossMaxHealth` 中统一修改。暂停界面也支持鼠标操作。报告中的 Boss 画面由当前 Release 程序和正式渲染函数生成，不是概念图。

验收点：暂停与继续都成功；Boss 截图清晰显示 `10/10` 血量。

### 03:05—04:05 面向对象三大特性与 STL

画面：切换到 `GameObjects.h`、`MoeBubbleGame.h` 或报告中的“主要类关系与 STL 容器”图。

讲解：

> 项目重点体现了封装、继承和多态。封装方面，地图格、玩家生命、速度、水泡容量等数据保持私有，只通过公开方法修改，以保证边界和上限。继承结构由 `GameObject` 到 `Character`，再分成 `Player` 和抽象的 `Enemy`；巡逻敌人、追踪敌人和 Boss 都继承 `Enemy`。多态方面，主循环使用 `vector<unique_ptr<Enemy>>` 保存不同派生对象，并统一调用更新、绘制、方向决策和受击虚函数，Boss 的多血量逻辑由重写版本处理。固定地图和键盘状态使用 `std::array`，水泡、水浪、道具、粒子和敌人使用 `std::vector`，对象清理由 `remove_if` 和 `erase` 完成，随机地图和 AI 使用 `std::mt19937`。

验收点：镜头能看清类名、虚函数和 STL 容器，不要快速滚动代码。

### 04:05—04:35 测试与总结

画面：展示报告中的测试表、四练习与五张游戏截图，最后回到通关画面。

讲解：

> 四个 EasyX 练习和游戏工程都采用面向对象结构。两个解决方案已经在 Debug、Release 和 x86、x64 共八组配置下编译通过。电子报告包含二十条测试用例，覆盖鼠标键盘、移动边界、连锁水浪、道具上限、Boss 血量、资源相对路径和通关结算。项目目前完成了三关玩法、完整 UI、原创角色与音频、差异化属性和报告材料。我的汇报到这里，谢谢老师。
