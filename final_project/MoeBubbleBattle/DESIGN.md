# 《萌泡大作战》项目设计基线

## 1. 项目定位

《萌泡大作战》是一款使用 C++、Visual Studio 和 EasyX 开发的原创 Q 版单人网格闯关游戏。玩法借鉴水泡爆炸类游戏，但不使用商业游戏的名称、角色、地图、图片或音频素材。游戏画面主要由 EasyX 基本图元绘制，原创角色设定图用于美术与报告参考。

玩家在迷宫中移动并放置水泡。水泡倒计时结束后向上下左右释放水浪，能够破坏木箱、触发其他水泡并淘汰敌人。玩家击败本关全部敌人后进入下一关，完成三关即胜利。

## 2. 必做功能

- 开始菜单、操作说明、游戏界面、暂停、过关、失败和通关界面。
- 三个递进关卡，敌人数量、移动速度和地图障碍逐步增强。
- 玩家四方向移动、网格与障碍碰撞、放置水泡。
- 水泡容量限制、2.5 秒倒计时、十字形水浪和连锁触发。
- 固定墙、可破坏木箱、空地三类地图单元。
- 敌人自动寻路/避险，接触水浪后被淘汰。
- 木箱被破坏后有概率掉落强化道具。
- 四类道具：水浪范围、同时放泡数量、移动速度、护盾。
- 分数、生命、当前关卡、剩余敌人和强化状态显示。
- 重新开始、返回菜单和安全退出。

## 3. 操作方式

| 操作 | 按键 |
| --- | --- |
| 移动 | `W/A/S/D` 或方向键 |
| 放置水泡 | `Space` |
| 暂停/继续 | `P` |
| 重新开始 | `R`（失败或通关界面） |
| 返回/退出 | `Esc` |

## 4. 游戏规则

- 玩家初始 3 点生命、水泡容量 1、水浪范围 2 格。
- 已放置的水泡在玩家离开其格子后成为不可穿越障碍。
- 水泡倒计时 2.5 秒，水浪沿四个方向传播。
- 固定墙完全阻挡水浪；木箱被破坏并终止该方向传播；其他水泡会立即被连锁触发。
- 玩家被水浪击中损失 1 点生命并短暂无敌；护盾可以抵消一次伤害。
- 敌人全部淘汰后进入下一关；玩家生命归零则游戏失败。
- 破坏木箱获得 20 分，淘汰敌人获得 300 分，通过关卡获得额外奖励。

## 5. 面向对象类设计

| 类 | 职责 |
| --- | --- |
| `MoeBubbleGame` | 初始化窗口、维护游戏循环、切换场景、组织各模块协作 |
| `InputManager` | 保存当前帧和上一帧键盘状态，区分按下与持续按住 |
| `GameMap` | 管理地图单元、障碍碰撞、木箱破坏和关卡生成 |
| `GameObject` | 可更新、可绘制对象的抽象基类 |
| `Character` | 玩家和敌人的共同基类，封装位置、速度、碰撞盒与存活状态 |
| `Player` | 处理输入、生命、强化状态和放泡能力 |
| `Enemy` | 敌人抽象基类，声明 AI 决策接口 |
| `PatrolEnemy` | 随机巡逻，在路口改变方向 |
| `HunterEnemy` | 根据玩家位置追踪，同时规避危险区域 |
| `WaterBubble` | 倒计时、爆炸范围、连锁触发和所有者容量归还 |
| `WaterWave` | 表示短暂存在的水浪格子并完成伤害判定 |
| `PowerUp` | 封装道具种类、移动/拾取状态和绘制 |
| `StarParticle` | 命中、破坏和过关时的粒子效果 |

其中 `GameObject`、`Character` 和 `Enemy` 使用虚函数体现运行时多态；`Player`、`PatrolEnemy`、`HunterEnemy` 通过继承复用角色能力；地图、对象容器和游戏状态由 `MoeBubbleGame` 组合管理。

## 6. 状态流程

`主菜单 → 操作说明/开始游戏 → 游戏中 ↔ 暂停 → 关卡完成 → 下一关 → 最终胜利`

游戏中玩家生命归零进入失败界面；失败或胜利界面可以重新开始，也可以返回主菜单。

## 7. 地图与界面

- 窗口尺寸：`960 × 720`。
- 地图：`15 × 13` 个格子，每格 `44 × 44` 像素。
- 左侧为游戏地图，右侧为分数、生命、关卡、敌人数量和操作提示。
- 固定墙、木箱、水泡、水浪、角色和道具采用不同高对比度颜色与 Q 版形状绘制。
- 玩家出生点附近预留空地，防止开局被木箱封死。

## 8. 验收重点

- Visual Studio 的 Debug/Release、Win32/x64 均能编译。
- 不存在绝对路径，不依赖本机图片或音频。
- 玩家不能穿墙或穿过已经离开的水泡格。
- 水浪被墙/木箱正确阻挡，连锁水泡只爆炸一次。
- 强化数值存在上限，连续按键不会无限生成水泡。
- 三关均可正常进入、胜利、失败、暂停和重开。
- 至少整理 12 个测试用例，其中包含地图边界、连续按键、连锁爆炸和生命归零等情况。

## 9. 后续可选增强

- 背景音乐和提示音。
- 更多敌人类型与随机地图种子。
- 排行榜或最高分本地保存。

这些增强不影响基础版本验收，只有在必做功能稳定后再加入。
